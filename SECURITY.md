# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest  | ✅        |
| Older   | ❌        |

Only the latest release receives security fixes.

## Reporting a Vulnerability

**Do not open a public GitHub issue for security vulnerabilities.**

Please report security issues by emailing the maintainer privately or using [GitHub's private vulnerability reporting](https://github.com/carloscosta2025086-web/WootFlow/security/advisories/new).

Include:
- A description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fix (optional)

You can expect an acknowledgement within 48 hours and a patch or mitigation plan within 14 days for confirmed issues.

## Scope

This project is a local desktop application. It:
- Does not collect or transmit any user data
- Does not connect to any external service
- Communicates only on `localhost` between the Python backend and the frontend UI
